Movie API App Goal: To build the server-side component of a “movies” web application. The web application will provide users with access to information about different movies, directors, and genres. Users will be able to sign up, update their personal information, and create a list of their favorite movies

URL endpoints

GET /movies Return a list of ALL movies to the user

GET /movies/:title Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user

GET /movies/genres/:genreName Return data about a genre (description) by name/title (e.g., “Thriller”)

GET /movies/:directors Return data about a director (bio, birth year, death year) by name.

GET /users Returns a list of all users and their details

POST /users Allow new users to register

example request body, in json:

{ "username":"testuser","password": "12345", "email": "testemail@email.com", "birthday":"1990-01-27" }

Mongoose Schema:

{ username: { type: String, required: true }, password: { type: String, required: true }, email: { type: String, required: true }, birthday: Date, favoriteMovies: [{ type: mongoose.Schema.Types.ObjectId, ref: "movies" }], }

Format of the response data: json. Returns newly added user.

PUT /users/:username Allow users to update their user info by username

example request body, in json:

{ "username":"updateduser","password": "12345", "email": "testemail@email.com", "birthday":"1990-01-27" }

No minimum number of properties that need to be passed.

Format of the response data: json. Returns newly updated user

POST /users/:username/movies/:movieID Allow users to add a movie to their list of favorites.

Format of the response data: json. Returns newly updated user.

DELETE /users/:username/movies/:movieID Allow users to remove a movie from their list of favorites. Format of the response data: json. Returns newly updated user.

DELETE users/:username Allow existing users to deregister

returns text saying which user was deleted "user ___ was removed"
